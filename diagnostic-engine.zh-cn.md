# 诊断引擎：组织同步稳定性™ 量化核心
## Diagnostic Engine: Organizational Sync-Stability™ Quantification Core

> **文件定位**
>
> 本文件是 `governance-operating-system.md` 的技术内核。
>
> 董事会看到的是信号。
> 引擎内部运行的是信号的生成逻辑。
>
> 此处定义 **如何从原始数据走向组织同步稳定性™ 得分**。
> 此处不涉及响应协议。响应协议由治理操作系统定义。

---

## 一、 引擎架构总览

原始数据流 → 变量标准化模块 → 公式计算模块 → 信号转换模块 → 阈值触发模块
↑ ↑ ↑ ↑ ↑
多源接入 统一量纲 核心算法 得分输出 治理信号


| 模块 | 功能 | 输出 |
| :--- | :--- | :--- |
| 变量标准化模块 | 将异构数据统一为可计算变量 | ER, PR, RI, Acceleration Pressure (标准化值) |
| 公式计算模块 | 执行核心公式运算 | 原始组织同步稳定性™ 值 |
| 信号转换模块 | 将原始值映射到标准化区间 | 0–100 标准化得分 |
| 阈值触发模块 | 对标准化得分施加趋势与阈值逻辑 | 治理信号（绿/黄/橙/红，趋势警报） |

## 二、 变量标准化模块

### 2.1 变量定义与数据来源

#### 执行节奏 (Execution Rhythm, ER)

| 维度 | 说明 |
| :--- | :--- |
| **定义** | 组织将输入转化为价值输出的结构性节奏能力 |
| **正向指标** | 交付周期稳定性、跨部门协同完成率、决策-执行闭环速度 |
| **负向指标** | 项目延期率、优先级频繁切换指数、未完成承诺堆积量 |
| **建议数据源** | 项目管理系统、OKR/KPI追踪系统、运营节奏元数据 |
| **标准化方向** | 0–1，越高表示执行节奏越健康 |

#### 压力调节 (Pressure Regulation, PR)

| 维度 | 说明 |
| :--- | :--- |
| **定义** | 组织感知、吸收、缓释内外部压力而不发生结构性变形的能力 |
| **正向指标** | 压力预警响应速度、资源弹性调配能力、冲突升级前的化解率 |
| **负向指标** | 关键节点过载频率、压力传导至绩效的时间缩短、压力信号被忽视的周期数 |
| **建议数据源** | 人力系统、运维监控、员工脉搏调查、事故报告时间线 |
| **标准化方向** | 0–1，越高表示压力调节能力越强 |

#### 恢复完整性 (Recovery Integrity, RI)

| 维度 | 说明 |
| :--- | :--- |
| **定义** | 组织恢复窗口的存在性、充足性与受保护程度 |
| **正向指标** | 恢复窗口的规律性、恢复期内无干扰比例、恢复后绩效回升幅度 |
| **负向指标** | 恢复窗口被压缩的频率、连续工作周期长度、休假实际执行率 |
| **建议数据源** | 工作强度数据、加班记录、会议密度趋势、恢复期合规日志 |
| **标准化方向** | 0–1，越高表示恢复能力越完整 |

#### 加速度压力 (Acceleration Pressure)

| 维度 | 说明 |
| :--- | :--- |
| **定义** | 作用于组织的内外部加速度合力 |
| **正向指标** | 市场节奏加速、竞争强度、内部变革频率与幅度、目标增长率 |
| **负向指标** | （本变量本身即是压力指标，负向指标即为压力超载信号） |
| **建议数据源** | 战略目标数据、市场情报、变革日志、资源饱和度监测 |
| **标准化方向** | 0–1，越高表示加速度压力越大（在公式中处于分母位置） |

### 2.2 标准化方法（推荐）

每个变量由若干子指标加权合成，原始值经归一化处理映射至 0–1 区间。

**通用归一化公式**（适用于有历史基准线的指标）：

标准化值 = (当前值 - 历史最低值) / (历史最高值 - 历史最低值)

对于反向指标（值越高越差），取补数：

标准化值 = 1 - [(当前值 - 历史最低值) / (历史最高值 - 历史最低值)]


**权重设定原则**：
- 首次部署时使用等权或专家赋权
- 运行四个季度后，基于每个子指标与组织稳定性事件的相关性进行统计校准

## 三、 公式计算模块

### 3.1 核心公式（计算层）

**组织同步稳定性™ (原始值) = (ER × PR × RI) / 加速度压力**

其中：

| 变量 | 值域 | 说明 |
| :--- | :--- | :--- |
| ER | 0–1 | 执行节奏标准化值 |
| PR | 0–1 | 压力调节标准化值 |
| RI | 0–1 | 恢复完整性标准化值 |
| 加速度压力 | 0.01–1 | 加速度压力标准化值（设定最小边界为0.01，避免分母为零） |

### 3.2 公式的数学直觉

- **分子是三变量之积**：任何一个变量崩溃，整体稳定性趋向于零。这体现了系统的串联脆弱性——执行、调节、恢复三者缺一不可。
- **分母是加速度压力**：当加速度压力趋近于零（组织静止），即使分子很小，稳定性得分也很高。这符合物理直觉——静止的系统不会失稳。
- **非线性特征**：当ER、PR、RI中任意一个接近零，乘积效应会放大崩溃，这与现实中组织突然失效的行为一致。

## 四、 信号转换模块

### 4.1 将原始值映射到 0–100 标准化得分

原始值理论上可在 0 到极大值之间浮动（当加速度压力极小时）。需要经平滑函数映射至标准化区间。

**推荐映射函数：对数压缩**

标准化得分 = 100 × [ln(1 + 原始值) / ln(1 + 原始值基准上限)]


**基准上限设定**：取部署后历史四个季度的最大原始值，或在首次部署时预设为 3.0（对应 ER=1, PR=1, RI=1, 加速度压力≈0.33）。

### 4.2 映射效果

| 原始值 | 近似标准化得分（基准上限=3.0） |
| :--- | :--- |
| 0.00 | 0 |
| 0.50 | ~29 |
| 1.00 | ~50 |
| 2.00 | ~79 |
| 3.00 | 100 |

**此映射确保得分在0–100之间，且具有直观可读性。**

## 五、 阈值触发模块

### 5.1 阈值区域定义

| 标准化得分 | 区域 | 信号 |
| :--- | :--- | :--- |
| 70–100 | 绿区 | 受控同步 |
| 50–69 | 黄区 | 波动扩展 |
| 30–49 | 橙区 | 结构性风险 |
| <30 | 红区 | 崩溃加速 |

---

### 5.2 趋势触发逻辑

趋势检测基于两个连续数据点（季度）的变化百分比：

Δ% = (当前得分 - 上期得分) / 上期得分 × 100%


| 条件 | 信号 |
| :--- | :--- |
| Δ% < -8% 且 > -15% | 黄色警报 |
| Δ% < -15% | 红色警报 |

---

### 5.3 触发输出格式

当阈值或趋势条件被满足时，引擎输出以下结构，供治理操作系统消费：

  "sync_stability_score": 47,

## 六、 校准与验证

### 6.1 初次部署校准

步骤	内容
1	设定各变量的子指标及初始权重
2	收集至少四个季度的历史数据作为基线
3	运行引擎生成回顾性得分
4	与组织已知的稳定性事件（重大失误、集体倦怠、关键人员流失等）进行对照
5	调整权重与映射参数，使引擎输出与已知事件高度吻合

---

### 6.2 持续校准

每四个季度审查一次权重

每次组织重大结构调整后重新校准

基准上限每年根据最新数据更新
